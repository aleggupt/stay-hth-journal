# stay-hth-journal

这是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或具体网站，仅作为页面文件的集中存放与历史版本记录。

## 项目简介

本仓库旨在提供一个简洁、稳定的 HTML 页面归档空间。  
所有页面均为独立构建，彼此之间不依赖外部服务或特定运行环境，可直接在浏览器中打开查看。

适用于：
- 个人笔记或日志的静态展示
- 临时页面的版本存档
- 轻量级前端实验或样板收集

## 目录说明

```
├── index.html          # 仓库首页（可选）
├── pages/              # 主要页面归档目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立 HTML 页面，文件名应反映内容或用途。
- `assets/`：存放页面共用的样式、脚本或图像资源，避免在页面内重复引用外部链接。

## 页面归档说明

- 每个页面均为自包含的 HTML 文件，可独立使用。
- 页面之间无强制导航或链接依赖，但可通过 `index.html` 提供索引。
- 归档内容不涉及任何特定平台、网站或商业用途，仅作技术存档。
- 文件命名建议使用小写字母、数字和连字符，避免空格和特殊字符。

## 维护说明

- 欢迎提交 Pull Request 添加新页面或改进现有页面。
- 新增页面请放入 `pages/` 目录，并在必要时更新 `assets/` 下的公共资源。
- 请勿在页面中嵌入跟踪脚本、广告或外部不可控资源。
- 所有页面应遵循基本的 HTML 规范，并确保在主流浏览器中可正常显示。
